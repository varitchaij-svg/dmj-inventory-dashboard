## ⚠️ Known Issues

- **updateOrderState row mapping** — คำนวณ `sheetRow = rowNum + 1` แต่ `readOrders_` สร้าง id แบบ `R{แถว}` ตรงๆ และ `deleteOrderRow` ลบแถวตรงๆ ทำให้การเขียน status อาจคลาดไป 1 แถว (ปัจจุบันรอดเพราะมี fallback จับคู่ด้วย sku+date) — ควรแก้ให้ `sheetRow = rowNum`
