2. conflict detection พังเพราะ cache บั๊กจริง

appsscript_complete.gs:1080–1092 · doGet:435 · cache:3115

lastModified ถูกคำนวณตอนสร้าง payload แล้ว cache ไว้ 180 วิ client เลยถือ epoch ที่เก่าได้ถึง 180 วิ ทำให้กรอบ +5000ms ไม่มีความหมาย

📍 2 คนเปิดจาก cache เดียวกัน → ทั้งคู่ถือ epoch เท่ากัน → เขียนทับกันได้ (false-pass) หรือ cache เก่ามาก → เขียนอะไรก็โดน reject (false-reject)
✅ แก้: คำนวณ lastModified สด ๆ (ห้าม cache) แล้วแยก shouldRejectForConflict(lastMod, clientLoadedAt, slop) เป็น pure function เทสต์ขอบที่ +5000 พอดี
