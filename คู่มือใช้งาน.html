🌸 ดูมึงจิง — คู่มือใช้งาน

ระบบจัดการสต็อกร้าน · ใช้บนมือถือ

เริ่มต้น ใครเห็นอะไร สั่งของ โอน/รับของ เช็คสต็อก นับสต็อก พิมพ์ป้าย งานจัดพิเศษ การ sync สิ่งที่ต้องเทสต์ ปัญหาที่เจอบ่อย

1. เริ่มต้นใช้งาน

  1. เปิดเว็บ dmj-inventory-dashboard.pages.dev บนมือถือ
  2. กด "Add to Home Screen" / "เพิ่มลงหน้าจอหลัก" เพื่อใช้เหมือนแอป (เปิดเร็ว ใช้ได้แม้เน็ตช้า)
  3. เลือก บทบาท ของตัวเอง (เจ้าของ / หน้าร้าน / คลัง / Sale)
  4. ถ้าเป็น เจ้าของ ต้องใส่ PIN · บทบาทอื่นกดเข้าได้เลย
💡 โหมดง่าย (สำหรับคนไม่ถนัดมือถือ) กดรูปโปรไฟล์มุมขวาบน → เปิด โหมดง่าย ระบบจะโชว์เฉพาะเมนูที่ใช้บ่อย 4 อัน ปุ่มใหญ่ กดง่ายขึ้น
🚪 ออกจากระบบ / เปลี่ยนบทบาท กดรูปโปรไฟล์มุมขวาบน แล้วกดปุ่ม 🚪 ออก

2. ใครเห็นเมนูอะไรบ้าง

👑 เจ้าของ เห็นทุกเมนู — ภาพรวม, ยอดขาย, สั่ง, โอน, สต็อก, นับสต็อก, รายงาน, Label ทุกอย่าง
🌸 หน้าร้าน สินค้า, สต็อก, เช็คหน้าร้าน, โอน, สั่งของ, งานจัดพิเศษ, Label
🏭 คลังสินค้า สินค้า, สต็อก, ตำแหน่งคลัง, นับสต็อก, รายการสั่ง, สรุปออกจากคลัง, Label
💼 Sale สินค้า, สต็อก, สั่งของ, งานจัดพิเศษ, Label
เมนูล่างของจอ = เมนูหลัก · ถ้าเมนูไม่ครบให้กด เพิ่มเติม ⋯

3. 🛒 สั่งของไปขาย

วิธีสั่ง (จากหน้า "สินค้า & สั่ง" หรือ "สต๊อก")

  1. เปิดแท็บ 🛍️ สินค้า & สั่ง หรือ ⚠️ สต๊อก
  2. หาสินค้าที่ต้องการ — พิมพ์ชื่อ/SKU ในช่องค้นหา (พิมพ์หลายคำได้ เช่น "แจกัน ขาว")
  3. กดปุ่ม 🛒 สั่ง ที่สินค้านั้น
  4. ใส่ จำนวน + เลือกประเภท (ปกติ / หิ้ว)
  5. กด ✅ ยืนยันสั่ง
เห็นป้าย "สั่งแล้ว N" บนสินค้า = มีคนสั่งของชิ้นนี้ค้างอยู่ N ชิ้น ตรวจก่อนสั่งซ้ำ จะได้ไม่สั่งเบิ้ล
⚠️ ของหมดสั่งไม่ได้ ถ้าสินค้าขึ้น "หมดสต๊อก" ปุ่มสั่งจะกดไม่ได้ ต้องรอของเข้าคลังก่อน

ดูรายการที่สั่งไปแล้ว

เปิดแท็บ 📋 รายการสั่งของ — เห็นทุกออเดอร์ที่ยังค้าง (สถานะ "รอ") คลังจะมาจัดให้

4. 🔄 โอนของ & รับของ (คลัง → หน้าร้าน)

ฝั่งคลัง — โอนของไปหน้าร้าน

  1. เปิดแท็บ ⚠️ สต๊อก (หรือ เช็คหน้าร้าน)
  2. หาสินค้าที่จะโอน → กดปุ่มโอน
  3. ใส่จำนวนที่โอน → ยืนยัน
  4. ระบบขึ้น Toast เขียว = โอนสำเร็จ · ตัวเลขคลังลดทันที

ฝั่งหน้าร้าน — รับของที่โอนมา

  1. เห็นแบนเนอร์ 📦 สินค้าที่โอนแล้ว รอรับ ในหน้าสต็อก
  2. เช็คของจริงให้ตรงกับจำนวน
  3. กด ยืนยันรับของ → ของจะเข้าสต็อกหน้าร้าน
⚠️ สำคัญ: ต้องกด "รับของ" เสมอ ถ้าโอนแล้วแต่หน้าร้านยังไม่กดรับ ระบบจะยังนับว่า "รอรับ" ตัวเลขหน้าร้านจะยังไม่อัปเดต

5. ⚠️ เช็คสต็อก & แจ้งเตือน

สี/สัญลักษณ์หมายความว่า
🔴 ต้องสั่งทันทีใกล้หมดมาก ต้องสั่งด่วน
🟡 ใกล้หมดเหลือน้อย เตรียมสั่ง
🛒 ควรสั่งหน้าร้านเหลือ ≤12 และคลังยังมีของ
คลัง / หน้าร้าน2 ตัวเลขแยกกัน — คลัง=สต็อกหลัง, หน้าร้าน=ของที่วางขาย
⚙️ ตั้งเกณฑ์แจ้งเตือนได้ เจ้าของตั้งได้ว่าเหลือกี่ชิ้นถึงจะเตือน (ในหน้าสต๊อก ด้านล่าง "เกณฑ์แจ้งเตือน")
🔔 LINE แจ้งเตือนอัตโนมัติ ระบบส่ง LINE ให้เจ้าของ: สต็อกใกล้หมด (วันละครั้ง), สรุปเช้า (เฉพาะวันมีออเดอร์ค้าง/งานพิเศษ), ออเดอร์หิ้วใหม่

6. 📊 นับสต็อกคลัง

  1. เปิดแท็บ 📊 นับ stock คลัง
  2. เดินนับของจริงในคลัง
  3. ใส่จำนวนที่นับได้ในแต่ละสินค้า
  4. กด ยืนยันการนับ → ระบบอัปเดตตัวเลขคลังให้ตรงของจริง
💡 ทำตอนปิดร้าน/ต้นเดือน เพื่อให้ตัวเลขในระบบตรงกับของจริง

7. 🖨️ พิมพ์ Label / ป้ายสินค้า

  1. เปิดแท็บ 🖨️ พิมพ์ Label
  2. เลือกสินค้าที่จะพิมพ์ป้าย
  3. กดพิมพ์ → ต่อเครื่องพิมพ์ฉลาก
🔄 sync ข้ามเครื่อง กดพิมพ์แล้วระบบจำว่า "พิมพ์แล้ว" — เครื่องอื่นจะเห็นว่าป้ายนี้พิมพ์ไปแล้ว ไม่พิมพ์ซ้ำ

8. 🎁 งานจัดพิเศษ (MTO)

งานจัดช่อ/จัดกระเช้า/งานสั่งทำพิเศษ ที่ใช้สินค้าหลายชิ้นรวมกัน

  1. เปิดแท็บ 🎁 งานจัดพิเศษ
  2. ดูงานที่ "กำลังจัด"
  3. จัดเสร็จ → กด ปิดงาน → ระบบตัดสต็อกวัตถุดิบที่ใช้ออกอัตโนมัติ + ส่ง LINE แจ้งเจ้าของ

9. 🔗 การเชื่อมต่อ & sync

ระบบทำอะไรความถี่
Google Sheetฐานข้อมูลหลัก ทุกอย่างเก็บที่นี่ทันที
ZORTดึงสต็อก/ยอดขาย/รูปสินค้า + ส่งสต็อกกลับทุก 2 ชม.
LINEแจ้งเตือนเจ้าของตามเหตุการณ์
⚠️ ตัวเลขอาจหน่วง ~2 ชม. ZORT sync ทุก 2 ชั่วโมง ถ้าแก้ใน ZORT ตรงๆ ตัวเลขในเว็บอาจยังไม่เปลี่ยนทันที — แต่ถ้าสั่ง/โอน/นับ ในเว็บ จะอัปเดตทันที
🛑 มีคนแก้พร้อมกัน 2 เครื่อง ระบบมีกันชนกัน (conflict check) ถ้าขึ้นเตือนว่าข้อมูลไม่ตรง ให้ รีเฟรชหน้า แล้วทำใหม่

✅ สิ่งที่ต้อง "ทดสอบจริง" ก่อนใช้เต็มตัว

ลองทำตามลิสต์นี้กับ ของจริง 1-2 ชิ้น แล้วเช็คว่าตัวเลขถูก:

🛒 ทดสอบการสั่งของ

  • สั่งของ 1 รายการ → ไปเช็คแท็บ "รายการสั่งของ" ว่าขึ้นจริง
  • เช็คว่า Google Sheet ชีต "ลำดับที่สั่งสินค้า" มีแถวใหม่
  • สั่งของชิ้นเดิมซ้ำ → ดูว่าขึ้นป้าย "สั่งแล้ว N" เตือนไหม
  • สั่งของ ประเภท "หิ้ว" → เช็คว่า LINE เด้ง แจ้งเจ้าของ (รอเดือน ก.ค. quota รีเซ็ตก่อน)

🔄 ทดสอบโอน & รับของ

  • โอนของจากคลัง 1 ชิ้น → ตัวเลขคลัง ลดทันที + ขึ้น Toast เขียว
  • ฝั่งหน้าร้านเห็นแบนเนอร์ "รอรับ" ไหม
  • กด รับของ → ตัวเลขหน้าร้าน เพิ่มขึ้น ตรงจำนวน
  • เช็คว่า ผลรวม (คลัง+หน้าร้าน) เท่าเดิม ก่อน/หลังโอน

📊 ทดสอบสต็อก & ZORT

  • นับสต็อก แก้ตัวเลข 1 ชิ้น → ยืนยัน → ดูว่าตัวเลขเปลี่ยน
  • รอ 2 ชั่วโมง (หรือสั่ง sync เอง) → เช็คว่าตัวเลข ไม่เด้งกลับ ค่าเดิม (นี่คือบั๊กที่เพิ่งแก้)
  • เทียบตัวเลขในเว็บ กับใน ZORT ว่าตรงกัน
  • ของที่ขายใน ZORT → สต็อกในเว็บลดตามไหม (หลัง sync)

🖼️ ทดสอบรูป & การแสดงผล

  • เปิดทุกแท็บ → รูปสินค้าขึ้นครบ ทุกหน้า (นี่คือสิ่งที่เพิ่งแก้)
  • กดรูปแล้ว ขยายดูได้ (lightbox)
  • ลองทุกบทบาท (เจ้าของ/หน้าร้าน/คลัง/Sale) → เมนูถูกต้อง
  • เปิด โหมดง่าย → เหลือ 4 เมนูหลัก ปุ่มใหญ่ขึ้น

🔔 ทดสอบ LINE (ต้นเดือน ก.ค.)

  • รัน debugLineMessage ใน GAS → LINE เด้งไหม (เช็คว่าหาย error 429)
  • ปล่อยให้ระบบส่ง "สรุปเช้า" → ได้วันละ ครั้งเดียว ไม่ spam
  • ตรวจปลายเดือน: ข้อความรวม ไม่เกิน 200

📱 ทดสอบกับคนใช้จริง

  • ให้ พนักงาน/แรงงานต่างด้าว ลองสั่ง+รับของเอง โดยไม่ต้องสอน
  • ดูว่า กดปุ่มถูก ไหม ปุ่มใหญ่พอไหม
  • เน็ตช้า/หลุด → เว็บยัง เปิดได้ ไหม (PWA offline)
