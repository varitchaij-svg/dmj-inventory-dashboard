🔒 Security & Audit Trail
ปิดช่องโหว่ + เพิ่มการบันทึกว่าใครทำอะไร ใหม่
  • Audit log ครบ 15/17 write action — deleteOrder(s)/deleteMtoJob/updateOrderState/createMtoJob/updateFrontStore/deleteLockEntry บันทึก actor + ค่าก่อน/หลัง (before/after) รูปแบบ JSON
  • writeAuditLog_ เป็น generic (รองรับทุก resource ไม่จำกัดแค่ SKU)
  • updateLockData บันทึก actor จริงแทน hardcode "ระบบ" + เพิ่มคอลัมน์ ผู้จัด (preparedBy) ในชีตรายการโอน
  • owner-only gate สำหรับ resetNegativeStock; เปิด TLS validation ใน imgProxy; เติม invalidateCache_() 6 จุดที่ขาด (conflict detection ครบ)
  • ตัดสินใจตาม ADR-001: ใช้แนวทาง KISS (เน้น traceability) แทน session-auth ที่ซับซ้อนเกินสเกลระบบ
⚡ Performance
เข้าเว็บครั้งแรกเร็วขึ้น + จอลื่นขึ้น ใหม่
  • lazy-load Recharts + defer qrcode/html5-qrcode — ไม่ block การโหลดครั้งแรก (กราฟโชว์ placeholder ระหว่างรอ ไม่ค้างจอ)
  • รูปสินค้าในลิสต์ใช้ loading=lazy — ลดภาระโหลดบนมือถือ
  • รวม filter ซ้ำใน CategoryView/OverviewView เป็น memo เดียว (ลด re-scan)
  • ลด SKU lookup ฝั่ง backend จาก O(items×rows) → O(items+rows) ในการโอน/นับสต็อกเป็นชุด
  • เชื่อม "จำนวนที่ควรสั่ง" (คำนวณจากยอดขาย) เข้าปุ่มสั่งของใน StockView
📷 Scanner & 🧪 Test
สแกนเร็วขึ้น + ระบบทดสอบอัตโนมัติ ใหม่

ตัวกรองสินค้าจม (Dead Stock Filter)
⏳ กรองสินค้าจมจากการโอนจริง ใหม่
  • ดึงประวัติการโอนจาก ZORT API (/Transfer/GetTransfers) ย้อนหลัง 2 ปี
  • กรองเฉพาะ คลังสาย5 (W0002) → หน้าร้าน (W0001) ที่ status Success
  • เก็บ "วันโอนล่าสุดต่อ SKU" ลงชีต "ประวัติโอนหน้าร้าน"
  • พบ 6,474 รายการ · 2,368 SKU จาก transfer ทั้งหมด 15,242 รายการ
  • Frontend คำนวณ deadMonths จากวันโอนล่าสุด (fallback: วันเข้าคลัง)
  • นับเฉพาะสินค้าที่ยังมีสต็อกในคลัง (qtyWH > 0) — ขายหมดแล้วไม่ถือว่าจม
  • แถบกรอง: เกณฑ์ 2 / 3 / 6 / 12 เดือน — แสดงเฉพาะระดับที่มีสินค้าเข้าข่ายจริง
⚠️ GAS ต้องทำ: copy โค้ดใหม่ → รัน syncTransferHistory → Deploy new version → ตั้ง trigger รายวัน

ZORT API Integration
📦 Purchase Order Sync ใหม่
  • ดึง PO จาก ZORT (/PurchaseOrder/GetPurchaseOrders) อัตโนมัติ
  • trigger รายสัปดาห์ (จันทร์ 06:00) — ไม่กิน quota เกิน
  • แก้ endpoint + field names จากการ explore API จริง
📊 ZORT Sales Sync ปรับปรุง แก้บัค
  • ดึงยอดขายย้อนหลัง 1 ปีเต็ม (เดิม 90 วัน) + รายวัน 60 วัน
  • กรอง order นอกช่วง (เช่น order ปี 2013) ออก ไม่ให้บิดสถิติ
  • หัว sheet เป็น text format กัน Google Sheets แปลง MM/YYYY เป็นวันที่
🖼️ ZORT Product Images ใหม่
  • sync รูปสินค้าจาก ZORT → col E ใน imageUrl sheet
  • ZORT เป็น primary source, manual (col D) เป็น fallback
  • trigger รายสัปดาห์ (จันทร์ 05:00)
🏭 Stock Push to ZORT แก้ไข แก้บัค
  • เปลี่ยนเป็น UpdateProductStockList (actual stock แทน available)
  • แก้ payload schema: stocks[].stock ตาม ZORT V4 จริง
  • warehousecode ส่งเป็น query param (ไม่ใช่ body)
